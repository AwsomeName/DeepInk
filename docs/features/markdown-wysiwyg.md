# Markdown S 级所见即所得编辑器

> 状态：产品设计已确认，尚未完成 S 级实现。
> 适用格式：`.md`、`.markdown`。
> 关联文档：`docs/features/document-editor.md`、`docs/features/file-type-support.md`。

## 产品结论

Markdown S 级定义为：

**安全的本地 Markdown 所见即所得编辑器，加上可携带源码行号发送给 Agent 会话的精确片段引用。**

目标不是覆盖所有 Markdown 方言，而是：

- 常用 Markdown 可以直接可视化编辑。
- 复杂语法至少可以安全打开和源码编辑。
- 不支持的内容不会在保存时被静默删除。
- 用户可以随时选取文档的一部分，连同文件位置和内容快照发送给当前会话。

S 级采用语义保真，不要求逐字符保真。允许等价标记规范化，例如 `*text*`
保存成 `_text_`；但标题层级、链接地址、图片、代码语言和原始内容块不得丢失。

## 已确认的产品决策

1. Markdown 使用所见即所得和源码两种模式，编辑同一份文档。
2. 自动保存只保存工作台恢复草稿，不自动覆盖磁盘文件。
3. Frontmatter 和 Mermaid 进入首轮；数学公式和脚注首轮只保证源码编辑。
4. 选区片段是下一条消息的一次性上下文，不是永久会话资源。
5. 本地插入图片默认复制到文档旁的 `.assets/<文档名>/`，Markdown 保存相对路径。

## 产品设计

### 编辑模式

#### 所见即所得

所见即所得是默认模式，面向日常写作。用户不需要输入 Markdown 标记即可编辑常用内容结构。

#### Markdown 源码

源码模式用于：

- 精细调整 Markdown。
- 编辑复杂或自定义语法。
- 检查保存前的规范化结果。
- 修复无法由所见即所得模式表达的内容。

两种模式共享同一个文档会话。切换模式本身不得：

- 清除撤销历史之外的文档状态。
- 改变 dirty 状态。
- 自动覆盖磁盘。
- 静默重写原文件。

首轮不提供常驻左右分栏。源码模式是保真出口，不是独立预览器。

### 语法支持分层

| 层级           | 语法                                                                                                                   | 产品行为                                                   |
| -------------- | ---------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| 原生可视化编辑 | 标题、段落、粗体、斜体、删除线、行内代码、链接、图片、有序列表、无序列表、任务列表、引用、分隔线、围栏代码块、GFM 表格 | 完整可视化编辑并保存为 Markdown                            |
| 专用内容块     | YAML Frontmatter、Mermaid                                                                                              | 所见即所得模式显示专用块；可编辑源码；Mermaid 同时渲染结果 |
| 原样保护       | 原始 HTML、未知 fenced block、自定义扩展语法                                                                           | 显示为原始 Markdown 块，保存时保留原始内容                 |
| 源码支持       | 数学公式、脚注、directives、MDX/JSX 等                                                                                 | 可以安全打开和源码编辑，首轮不承诺专用可视化交互           |
| 安全拒绝       | 无法稳定解析或序列化的异常文档                                                                                         | 提示进入源码模式；存在内容丢失风险时禁止普通保存           |

“原样保护”不代表逐字符保留整个文档，只承诺被识别为保护块的原始内容不被所见即所得序列化器删除或改写。

### 常用编辑能力

首轮完整交互包括：

- H1-H6 标题和正文。
- 粗体、斜体、删除线、行内代码。
- 有序、无序、嵌套和任务列表。
- 引用、分隔线和围栏代码块语言选择。
- 链接插入、编辑和移除。
- 图片插入、替代文本、标题和地址编辑。
- GFM 表格插入、增删行列和基础单元格编辑。
- 撤销、重做和常用键盘操作。

首轮不支持：

- Word 式字体、字号、颜色和分页。
- 图片裁剪、浮动环绕和任意像素定位。
- 高级表格公式、排序和合并单元格。

### 本地图片

支持相对路径、工作区内绝对路径和网络图片。

用户通过拖入、粘贴或文件选择器插入本地图片时：

1. 图片默认复制到文档旁的 `.assets/<文档名>/`。
2. 文件名冲突时自动生成不覆盖现有文件的名称。
3. Markdown 中写入相对路径。
4. 渲染进程通过受控本地资源接口加载图片，不直接访问 Node.js 文件系统。

未命名草稿在首次插入本地图片时，应先要求保存 Markdown，或明确保存到工作区草稿资源目录；首轮优先采用“先保存文档”。

### 保存与恢复

- `Cmd/Ctrl+S` 明确保存磁盘文件。
- 编辑过程持续镜像工作台恢复草稿。
- 恢复草稿不等于自动覆盖原文件。
- 保存前比较文件版本或内容指纹，检测外部修改。
- 外部冲突提供重新载入、查看源码差异、另存为和确认覆盖。
- 写入失败必须在界面中可见，并保持 dirty 状态。
- 磁盘写入采用临时文件和原子替换，降低中断造成的文件损坏。

## 选区发送到会话

### 产品定位

Markdown 选区是下一条消息的一次性上下文。它与文件、文件夹等长期挂载资源不同：

- 片段只随下一条用户消息发送。
- 发送成功后从待发送区移除。
- 已发送消息保留不可变引用快照。
- 后续消息不会自动重复携带旧片段。

### 用户流程

1. 用户在所见即所得或源码模式选中内容。
2. 浮动工具条和右键菜单显示“发送到会话”。
3. 片段进入当前会话输入区，显示为 `README.md L12-L24`。
4. 会话输入框获得焦点，但系统不自动发送消息。
5. 用户可以继续选择其他片段，或输入问题后发送。
6. 点击已挂载或已发送的片段，可跳回文档并定位范围。

快捷键作为工作台命令统一注册。首轮不默认占用 `Cmd/Ctrl+L`，避免与浏览器地址栏和现有工作台命令冲突。

### 片段数据

片段资源类型建议为 `file-range`，至少包含：

```ts
interface MarkdownFileRangeRef {
  type: 'file-range'
  format: 'markdown'
  path?: string
  tabId: string
  startLine: number
  endLine: number
  startColumn: number
  endColumn: number
  selectedText: string
  sourceSnapshot: string
  snapshotHash: string
  dirty: boolean
}
```

- `selectedText` 是用户精确选中的可见文字或源码。
- `sourceSnapshot` 是与选区相交的完整 Markdown 源码行。
- 行列号基于选取瞬间的当前编辑器缓冲区，不基于磁盘旧版本。
- 未命名草稿使用 `tabId` 定位。
- 已创建的片段是不可变快照，不随文档后续编辑更新。

### 选区映射

Tiptap 使用 ProseMirror 文档位置，Markdown 使用源码行列。不能通过搜索选中文字反推行号，因为重复文字、列表、链接和表格都会产生歧义。

文档内核需要提供带源码映射的序列化结果：

```text
ProseMirror 文档
    ↓
serializeWithSourceMap()
    ├── 规范化 Markdown
    └── ProseMirror position → Markdown line/column
```

首轮采用块级精确规则：

- 普通段落携带所有相交源码行和精确选中文字。
- 列表、引用、表格和代码块携带所有相交的完整源码行。
- Frontmatter、Mermaid 和原始 Markdown 块被部分选中时，携带整个块。
- 源码模式直接使用源码编辑器选区计算行列。

后续再扩展到完整的行内标记位置映射。

### 限制和失败路径

- 单片段最多 200 行或 32KB。
- 单条消息的全部片段最多 64KB。
- 超限时提示缩小选区或挂载整个文件。
- 文件删除后仍可发送快照，但不能跳转。
- 文档变化后，跳转时优先使用内容指纹重新定位。
- 无法唯一定位时打开文件，并提示该引用来自旧版本。
- 片段内容在 Agent 上下文中标记为用户引用材料，不作为系统指令。

## 功能边界

### 本轮包含

- 常用 Markdown 所见即所得编辑。
- Markdown 源码模式。
- Frontmatter、Mermaid 和未知语法保护。
- 本地图片插入和相对路径管理。
- 表格、链接、图片的完整插入交互。
- 手动磁盘保存、草稿恢复、保存冲突和错误反馈。
- 所见即所得和源码模式选区带行号发送到会话。
- Agent 读取当前完整 Markdown。

### 本轮不包含

- MDX/JSX 可视化编辑。
- 数学公式和脚注的专用可视化编辑。
- 多人协作、云同步、评论和修订。
- 文档版本历史。
- Word 式分页和自由排版。
- Agent 对任意选区的自动增量改写。
- Markdown 与 DOCX/PPTX 的双向转换。

## 架构边界

S 级实现不继续把能力堆叠在单个 React 组件中，至少拆分为：

```text
MarkdownDocumentSession
├── 原始源码、当前源码和 dirty
├── ProseMirror 文档
├── 文件版本和保存冲突状态
├── 语法诊断
└── 选区与源码位置映射

MarkdownCodec
├── parse()
├── serializeWithSourceMap()
├── roundTripDiagnostics()
└── 保护块识别

MarkdownEditorSurface
├── 所见即所得模式
├── 源码模式
└── 选区操作

MarkdownFileService
├── 文件读取和版本指纹
├── 草稿镜像
├── 原子保存
└── 本地图片资源管理
```

渲染进程不得直接使用 Node.js 或文件系统 API。文件读取、原子写入、图片复制和版本检查继续通过安全 IPC 进入主进程。

## 开发里程碑

### M1：文档内核与保真基线

目标：

- 建立可信的 Markdown 文档会话。
- 在继续扩展 UI 前，先证明常用语法可以安全往返。

方案：

- 引入 `MarkdownDocumentSession` 和 `MarkdownCodec`。
- 保存原始源码、规范化源码、ProseMirror 文档、dirty、诊断和文件版本。
- 建立语法语料库和解析、序列化、二次解析测试。
- 识别不支持或可能丢失的语法。
- 生成 ProseMirror 位置到 Markdown 源码的块级映射。

验收标准：

- 原生支持语法经过“源码 → 文档 → 源码 → 文档”后语义一致。
- 链接地址、图片地址、代码语言、列表层级和表格内容不丢失。
- Frontmatter、未知 fenced block 和原始 HTML 能被识别为保护内容。
- 存在丢失风险时产生明确诊断，不允许无提示继续保存。
- 语料测试覆盖中文、英文、重复文本、嵌套列表、表格、代码块和混合文档。

### M2：双模式与保护块

目标：

- 让任何 Markdown 都有安全编辑路径。

方案：

- 增加 Markdown 源码编辑模式。
- 实现 Frontmatter、Mermaid 和原始 Markdown 块节点。
- 两种模式共享同一文档会话和保存状态。
- 模式切换前运行序列化诊断。

验收标准：

- 所见即所得与源码模式可反复切换，文档语义不丢失。
- Frontmatter 和未知语法块的原始内容保持不变。
- Mermaid 可以显示渲染结果并编辑块源码。
- 无法安全进入所见即所得模式的文档自动进入源码模式并说明原因。
- 模式切换不会自动写盘或错误清除 dirty。

### M3：常用所见即所得交互

目标：

- 完成不依赖手写 Markdown 的日常创作闭环。

方案：

- 补齐链接、图片、表格、代码语言、撤销和重做交互。
- 接入本地图片选择、拖入、粘贴、复制和相对路径管理。
- 统一工具栏、浮动工具条和键盘命令。
- 消费编辑器字体、字号和换行等已有本地设置。

验收标准：

- 用户无需进入源码模式即可创建和编辑全部原生支持结构。
- 本地图片保存、关闭、重启后仍能正确显示。
- 图片重名不会覆盖现有资源。
- 链接、表格和图片操作均可撤销和重做。
- 工具栏状态与当前选区一致，保存失败有可见反馈。

### M4：选区带行号发送到会话

目标：

- 实现类似代码编辑器的精确上下文引用。

方案：

- 新增消息级 `file-range` 资源。
- 使用源码映射计算所见即所得选区的 Markdown 行列。
- 源码模式直接计算源码行列。
- 在编辑器浮动工具条、右键菜单和命令系统中提供发送动作。
- 在会话输入区显示待发送片段，消息发送后清理。
- 支持点击片段跳转回文档。

验收标准：

- 所见即所得和源码模式都能发送选区。
- 普通段落、重复文本、列表、表格、引用和代码块的行号正确。
- 未保存文档显示并发送当前内存快照。
- 可以在一条消息中附加多个片段。
- 片段不会自动进入之后的消息。
- 文档修改或删除后，旧快照仍可发送；无法跳转时有明确提示。
- 大片段和总量限制在界面、协议和主进程三层均生效。

### M5：保存、冲突和恢复安全

目标：

- 防止丢稿、文件损坏和无提示覆盖外部修改。

方案：

- 保存文件版本和内容指纹。
- 监听工作区文件变化并区分自身写入与外部写入。
- 实现临时文件加原子替换。
- 完善恢复草稿、冲突处理和错误提示。

验收标准：

- 应用异常退出后能恢复未保存草稿。
- 外部程序修改文件后，Studio 不会无提示覆盖。
- 保存失败后编辑内容和 dirty 状态仍保留。
- 冲突场景可以重新载入、查看差异、另存为或确认覆盖。
- 中断写入不会留下空文件或半写文件。

### M6：S 级综合验收

目标：

- 将 Markdown 从“基础富文本编辑器”提升为可发布的 S 级能力。

方案：

- 完成长文档性能、键盘、可访问性和异常语料检查。
- 联调 Agent 读取、保存和片段上下文。
- 删除重复或过时文档，统一事实源。

验收标准：

- 产品范围内不存在已知的静默内容丢失路径。
- 规定语料和核心用户流程全部有自动化回归测试。
- 大文档达到约定的打开、输入和模式切换性能基线。
- Markdown 片段可以被 Agent 准确识别为引用内容。
- 产品文档、文件支持分档和实际实现状态一致。

## 当前完成度

当前已有：

- Tiptap 和 Markdown 扩展。
- 常用块与格式扩展。
- 本地文件读取、手动保存和 dirty 状态。
- 工作台恢复草稿。
- Agent 全文读取、覆盖、追加和保存。

当前尚未达到 S 级，因为仍缺少：

- 系统化往返保真和风险诊断。
- 源码模式和保护块。
- 完整的链接、图片和表格交互。
- 选区源码映射和会话片段协议。
- 外部修改冲突和原子保存。
- 覆盖产品语料的编辑器级测试。
