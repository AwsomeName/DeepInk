# 平台自动化 — 路径选型与合规边界

> 本文档界定 CCLink Studio Agent 操作各类外部平台(电商 / 内容 / IM)时的**路径选择、合规边界与禁区**。
>
> 当前边界：本文作为本地浏览器、真机设备和 Agent 自动化合规参考；官方账号和消息网络由 CCLink 官方侧接入。

## 概述

CCLink Studio Agent 要替用户操作各种平台,存在两条路径:

| 路径 | 机制 | 合规 | 稳定性 | 适用 |
|------|------|------|--------|------|
| **官方 API / 开放平台** | 平台提供的 OpenAPI、Webhook、SDK | ✅ 完全合规 | ✅ 高(契约稳定) | 有开放能力的平台 |
| **UI 操控(真机设备)** | ADB 输入、截图和语义 UI 分析 | ⚠️ 看平台(可能违反 ToS) | ⚠️ 低(客户端更新即失效) | 无 API 的低风险平台 |

**核心原则:API 优先。** 有官方 API 的平台,绝不用 UI 操控;UI 操控是「无 API 时的下策」,且必须避开高风险平台。

## 平台合规地图

| 平台 | 类型 | 官方 API | UI 操控封号风险 | CCLink Studio 路径 |
|------|------|----------|-----------------|--------------|
| 京东 | 电商 | ✅ 京东开放平台 | 低(浏览/搜索) | API(商品/订单);设备 UI 仅做无 API 的浏览采集 |
| B 站 | 内容 | ✅ 开放平台(投稿等) | 中(自动化发布检测) | API(投稿/数据);UI 仅兜底 |
| **钉钉** | 企业 IM | ✅ **极完善**(机器人/服务端/Stream) | 低-中 | **走开放平台 API**,不需云手机 |
| **企业微信** | 企业 IM | ✅ 完善(企微开放平台) | 低-中 | **走 API**,不需云手机 |
| **飞书** | 企业 IM | ✅ 完善(Lark/Feishu Open Platform) | 低-中 | **走 API**,不需云手机 |
| **微信(个人号)** | 个人社交 IM | ❌ 无 | **极高(必封)** | **禁区**;走腾讯 IM SDK 自建 |
| **QQ(个人)** | 个人社交 IM | ❌ 无 | 极高 | 禁区 |

### 通用规律

- **企业级协作平台**(钉钉/企微/飞书)→ 把开放 API 当核心竞争力,**自动化的合规主场就在 API**。
- **个人社交 IM**(微信/QQ)→ 刻意封锁自动化,**UI 操控必踩风控,是禁区**。
- **电商/内容平台**(京东/B 站)→ 多有开放平台,**API 优先**;设备 UI 只做无 API 的浏览/采集。

## 分层策略

CCLink Studio 的 Agent 工具体系按此分层(MCP 工具模块):

1. **官方 API 类(优先,默认)** —— 各平台开放平台 SDK 封装为 MCP 工具
   - 钉钉 / 企业微信 / 飞书:消息收发、群管理、机器人
   - 京东 / B 站:商品 / 投稿 / 数据
   - 特点:合规、稳定、契约化,客户端更新不影响

2. **UI 操控类(兜底,仅限无 API + 低风险平台)** —— 用户自有真机 + ADB
   - 仅用于**浏览 / 搜索 / 采集**等低风险、非登录态敏感动作
   - **绝不碰个人社交 IM(微信/QQ)、绝不碰批量多账号营销**

3. **官方通道(无 API 的高敏感场景)** —— CCLink 官方消息网络
   - 微信场景的合规替代应走官方消息网络，不操控个人微信号客户端；Studio 默认不内置官方消息 SDK。

| 平台 | 开放平台入口 | 关键能力(Agent 自动化相关) |
|------|-------------|------------------------------|
| 钉钉 | [open.dingtalk.com](https://open.dingtalk.com) | 自定义机器人 Webhook、企业内部机器人(收发消息)、服务端 API、**Stream 模式(无需公网 IP 接收事件,适合桌面端)** |
| 企业微信 | [open.work.weixin.qq.com](https://open.work.weixin.qq.com) | 企微开放平台:消息、客户联系、应用 API |
| 飞书 | [open.feishu.cn](https://open.feishu.cn) | Lark/Feishu Open Platform:消息、机器人、多维表、事件订阅 |
| 京东 | [open.jd.com](https://open.jd.com) | 京东开放平台:商品、订单、物流 |
| B 站 | 搜索「bilibili 开放平台」 | 投稿、内容、数据 |
| CCLink 官方消息网络 | 由官方 CCLink 项目侧维护 | Studio 默认不内置官方消息 SDK、message credential 或消息路由 |

> ⚠️ 上述入口为平台通用开放平台,具体 API 能力 / 配额 / 资质要求(企业认证等)以各平台实时文档为准。本文档为路径选型指导,不替代各平台 API 详细接入设计。

## 设备 UI 在平台自动化中的定位

- ✅ 适合:无官方 API 的低风险平台的**浏览 / 采集 / UI 操作**
- ✅ 适合:用户自有真机上的端到端 App 任务执行
- ❌ 禁止:个人社交 IM(微信/QQ)操控
- ❌ 不必要:钉钉/企微/飞书等有完善 API 的平台(直接用 API)

## 决策清单(Agent 选路径时)

判断一个平台任务该走哪条路径:

1. **该平台有官方开放 API 吗?**
   - 有 → 用 API(钉钉/企微/飞书/京东/B 站)。**结束。**
   - 无 → 继续。
2. **该平台是个人社交 IM(微信/QQ)吗?**
   - 是 → **禁区**,不提供 UI 操控工具；如需消息能力，应由 CCLink 官方消息网络承接。
   - 否 → 继续。
3. **该任务是低风险浏览/采集,还是登录态敏感/批量营销?**
   - 低风险浏览/采集 → 用户自有真机 UI 操控兜底。
   - 登录态敏感/批量营销 → 评估合规风险,默认不提供,必要时人工确认。
